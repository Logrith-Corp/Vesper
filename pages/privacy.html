Privacy is

Every design decision in Vesper was made with one constraint: zero exposure by default. No toggles. No settings. Just structural privacy.

Our Position

Privacy tools are built wrong. They ask users to opt in, to enable, to configure. We believe that is a design failure.

True privacy means the system is structurally incapable of violating it — regardless of settings, updates, or user error.

No Data Minimization

We don't minimize data collection. We eliminate it. There is no pipeline to minimize from.

No Privacy Mode

There is no "private mode" in Vesper because all modes are private. It's the only mode that exists.

Verify Don't Trust

The privacy engine is open source. Don't take our word for it — read the code, run it, audit it yourself.

Technical Architecture

How we make it
structurally impossible
to violate your privacy.

User Interface
Fully Local
All rendering and UI logic runs locally. No remote frames, no third-party scripts injected into browsing context.
AI / Inference Layer
On-Device Only
All LLM inference runs on local hardware via quantized models. No API calls to OpenAI, Anthropic, or any external AI provider.
Network Layer
No Telemetry
Vesper makes no background connections to our servers. Zero heartbeats, zero analytics pings, zero crash reporting (unless explicitly opted in).
Storage Layer
Local + E2E Encrypted
Browsing history, bookmarks, and session data live on your device. Cloud sync uses client-side AES-256 encryption — we hold no decryption keys.
Update Mechanism
Signed Binaries
Updates are cryptographically signed and verified locally. Update server receives no identifiable information during the check process.
Four Guarantees

What we promise.
What you can verify.

⬡

On-Device Everything

Every computation — search indexing, AI inference, tab prediction, form autofill — happens on your hardware. We have no servers that receive your browsing data because no such infrastructure exists.

◈

Zero Log Architecture

Vesper maintains no logs of your browsing sessions on any external system. History stays local, encrypted at rest, and is never indexed or transmitted. Deleting it locally is final.

◉

Open Source Core

The privacy-critical components of Vesper — the network layer, telemetry system (or lack thereof), and sync encryption — are fully open source and available on GitHub. Audit them yourself.

⬖

E2E Encrypted Sync

If you use Vesper Sync, your data is encrypted on your device before it ever leaves. We use your passphrase-derived key. We store ciphertext only. A court order cannot help us decrypt your data.

Privacy FAQ

Hard questions.
Direct answers.

Can you access my browsing data?
No. Architecturally, we cannot. No telemetry pipeline exists. Our servers never receive browsing session data. This is verifiable in the open source codebase.
What happens if Vesper gets acquired?
The privacy architecture is structural, not policy-based. An acquirer cannot retroactively add telemetry to existing installations. The open-source core is MIT licensed.
How does AI work without sending data to OpenAI?
Vesper ships with quantized local models (3B–7B parameters) optimized for Apple Silicon and modern x86. No external API calls are made for any AI feature.
What if I lose my sync passphrase?
Your data is unrecoverable if you lose the passphrase. This is by design. We cannot reset it because we do not hold it. Keep it somewhere safe.
Is Vesper compliant with GDPR / CCPA?
Vesper exceeds GDPR and CCPA requirements by collecting no personal data at all. There is nothing to delete, export, or rectify because we hold nothing.
